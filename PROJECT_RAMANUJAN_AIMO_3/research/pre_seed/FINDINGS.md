# Pre-Seed Prompting — Findings

## Technique
Inject text into the model's analysis (hidden reasoning) channel before generation begins, making the model believe it started thinking in a particular direction.

## Seeds Tested
1. "Try small concrete cases before generalizing" — Steered toward brute-force exploration, caused timeouts
2. "Identify the core mathematical structure" — No measurable effect
3. "Decompose into subproblems, find the key constraint" — No measurable effect

## Results
- Prefill seeding showed zero positive signal across all experiments
- On AIME i_11: prefill caused 7/8 timeouts (vs 7/8 correct in baseline) by steering toward exhaustive computation
- On hard problems: identical 4/7 across all configs

## Root Cause
RL-trained reasoning models develop "template collapse" (RAGEN-2, arXiv 2604.06268) — fixed reasoning patterns learned through reinforcement learning that resist external steering. The first token of reasoning acts as a mode switch (from steering vector research), but generic seeds don't qualify as effective mode switches.

## What Might Work Instead
Problem-specific prefill from a critic model (community-reported success on DeepSeek R1). The seed would be generated by analyzing the specific problem, not generic across all problems.
